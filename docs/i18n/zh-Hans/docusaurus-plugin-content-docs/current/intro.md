# Cleanr 概览

Cleanr 帮你找出并移除**可以重新生成的开发者缓存**，不必把磁盘清理变成一次
冒险。它运行在终端中，会解释每个条目为什么被匹配，并由你决定哪些内容进入
系统回收站。

常见候选项包括：

- `node_modules` 等项目依赖；
- Rust `target`、Xcode `DerivedData` 等构建产物；
- Cargo、npm、pnpm、pip、Gradle 等工具的包缓存；
- 需要人工确认的大文件、日志和临时文件。

## 基本流程

Cleanr 将“发现”与“删除”明确分开：

1. **扫描**一个或多个目录。
2. **审阅**匹配到的候选项和风险说明。
3. **选择**真正想移除的条目。
4. **确认**清理。
5. 必要时**恢复**历史清理运行。

文件不会仅仅因为被发现就自动清理。高置信度、可重建的内容可能会被预选，
但执行前仍然可以修改计划。

## 安全机制概览

- 清理会把条目移动到系统回收站，不会永久删除。
- 计算可释放空间前会消除父子目录重叠。
- 执行前会再次检查目标；发生变化的文件、符号链接、扫描根目录外的路径和
  Cleanr 受保护数据都会被拒绝。
- 每次清理和恢复都会写入本地清单，便于检查结果。
- 插件默认是声明式数据文件；动态 hook 需要显式信任。

具体保证和恢复限制见[安全与恢复](./safety-and-recovery)。

## Cleanr 适合你吗？

Cleanr 面向希望通过键盘界面审阅生成文件和缓存的开发者。它不是通用系统
加速器、注册表清理工具，也不是无人值守的自动删除服务。
